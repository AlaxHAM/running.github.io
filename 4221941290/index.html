模块

​ 简单的理解，就是一堆可以调用的代码集合，一个 py 文件可以是一个模块被导入，一个模块（包）也可以是由多个 py 文件组成

​ 大致上模块可以分为三种：自定义模块；内置模块；开源模块

​ 自定义模块很好理解，即使程序员根据特殊需求和场景所定义的模块；开源模块就是需要通过 pip 下载的，有其他的程序员所构建的模块

pip

​ pip 是 python 的包管理工具，提供查看、下载、安装、卸载的功能，在 py2 中使用 pip，py3 中则是使用 pip3，但是使用 pip 去安装也不会有问题，只是推荐是 pip3

Code

​ 使用 pip 安装模块通常使用 pip install django，但是默认情况是去安装最新的版本，有时候通常会是要安装指定的版本

Code

​ 另外，安装默认去国外的资源下载安装，所以速度容易受影响，可能会出现一些超时或是无法下载，导致安装失败

​ 所以，可以使用国内的源去下载一些包，大部分都会含有最新的版本，如果只是临时安装一个包，使用 -i 参数可以指定国内的镜像源

Code

​ 除了使用参数，临时指定下载的源，也可以对配置做修改，实现永久的下载线路更改。linux 中找到 ~/.pip/pip.conf 的配置文件 pip.conf，如果文件不存在可以自己创建，然后编辑内容

Code

​ windows 下，需要在 user 目录下创建 pip 目录，并创建一个 pip.ini 文件，内容与上方一致

​ 卸载一个包，使用 pip uninstall Django，搜索包，使用 pip search Django

​ 有时，通常会需要根据给定的包配置去安装环境，或是要将当前环境使用的包配置打包，pip 的包配置文件是 requirement.txt 的文本文件，内容类似

Code

​ 如果知道了包配置文件，需要安装，通过参数 -r 并给出 requirement.txt 的目录，比如 pip install -r ./requirements.txt

​ 如果要根据当前环境，生成一个包配置文件，使用 pip freeze > ./requirements.txt

内置模块

​ 包括时间模块，random 模块，os 模块，sys 模块，序列化模块，re 模块，collocations 模块等等

time 模块

​ 对于 时间的表示有三种形式：时间戳，时间字符串 以及 结构化时间

​ 时间戳是纯数字的形式，时间字符串则是含有符号的表示的字符串，结构化时间是一个对象，将各个值赋给对应的属性

python

os 模块

python

​ os 模块中如果要删除一个目录，这个目录不是一个空目录的话，是会抛出 OSError 的异常的，也就说模块的 rmdir 以及 removedirs 通常都不能直接删除目录，因此可以通过递归的方法来判断，先将目录的文件删除，再将目录删除

​ 除此之外，还可以用 shutil 模块的 rmtree 的方法将一个路径的目录直接删除 shutil.rmtree(path)

sys 模块

​ 需要注点的一点是，os 模块所操作的是对于操作系统的，而 sys 模块则是用来操作 python 解释器进行交互的接口模块

python

序列化模块

​ python 中有两个用来序列化得到字符串的模块分别是：json 和 pickle，json 是一个数据传输的一个标准类型，而 pickle 只能是对 python 自己的类型和特性去做序列化，对比 json 来说支持的范围有限

python

hashlib 模块

​ 利用 hash 算法来对字符串进行编码操作，例如对登录用户的密码加密等，hashlib 中整合了 md5 和 sha，支持多种不同算法，编码的操作是不可逆的，无法通过编码后的数据来还原成原来的加密前的内容

python

​ 注意，使用普通的方式去进行编码，是能够被破解的，可以在一些破解网站中得到原字符串

​ 除此之外，编码的对象中还有一种 “加盐” 的方法，即在对象中加入字符再去做编码，这样的方式要比纯用普通方式加密安全很多，只要特定的密文不被泄露，基本就不可以被破解

python

logging 模块

​ 默认情况下 Python 的 logging 模块将日志打印到了标准输出中，且只显示了大于等于 WARNING 级别的日志，因为日志记录级别默认是设置为 WARNING

​ 日志级别等级：CRITICAL > ERROR > WARNING > INFO > DEBUG

python

​ 通过 logging.basicConfig() 来实现日志创建，通过参数对其进行设置

python

dis 模块

​ dis — Disassembler for Python bytecode，即把 python 代码反汇编为字节码指令；python 代码是先被编译为字节码后，再由 python 虚拟机来执行字节码， python 的字节码是一种类似汇编指令的中间语言， 一个 python 语句会对应若干字节码指令，虚拟机一条一条执行字节码指令， 从而完成程序执行

python

collections 模块

​ 是一个实现特定的目标容器，提供对 python 内置的数据结构上的补充

补充结构 说明
namedtuple() 可以命名的元组，元素的访问可以通过访问属性获取
deque() 双向对列，即对列的左右都可以进行增删操作
OrderedDict() 有序的字典结构，会按照 键-值 插入的顺序来返回，字典的子类
defaultdict() 给创建的字典附上默认的值，字典的子类
Counter() 用来统计集合中每一个元素的次数，字典的子类
python

mmap 模块

​ 是一个内存映射模块，用来提升对大本文的内容处理，其本质是将文件或是其他对象映射到进程的地址空间，来实现文件所在磁盘的地址与进程虚拟地址空间中的一块虚拟地址进行对映关系，相对于频繁要去读或写的大文件而言，利用 mmap 方法处理就显得更加高效且优雅，因为文件不会真正的被移入到内存中，而是利用虚拟地址关联对文件进行操作

python

流式上传

​ Requests 支持 流式上传，来 允许大的数据流或文件上传，且无需先将文件的数据全部读取到内存中

Session

​ 在 requests 中，直接使用 get 或 post 等方法可以模拟网页的请求，但实际上是相当于不同的 Session，也就是出相当于每次都打开一个新的浏览器并访问对应的页面，而不是在一个浏览器中打开多个不同的标签访问页面，当然可以在之后的请求中每次访问都加上获得的 cookie，但这样会略显麻烦。

​ 对此使用 Session，则相当于是维持了一个浏览器，这样就不用担心 cookie 的问题了

SSL证书验证

​ 这是一个针对 https 协议产生的一个验证方式，但是有时候某些网站可能会因为没有设置好 https 的证书或是证书不被 CA 认证机构认可，此时访问对应的页面则会出现一个 SSL 证书的错误提示，但是这样页面并不是不可以访问的

​ 对于这样的页面，直接使用 requests 去 get 网页的 url 的时候，requests 是会抛出一个 SSLError 的错误，来告知请求的 url 的证书是无效的，如果必须要爬取这个页面，可以通过 verify 的参数指定 False 来控制不对证书进行验证，默认情况下该参数的值是 True

​ 当然也可以通过证书文件的路径来指定认证证书（如果 verify 设为文件夹路径，文件夹必须通过 OpenSSL 提供的 c_rehash 工具处理），并且也可以通过 Session 将认证证书保持在会话中

python

​ 注意， 通过 verify 指定 False 后，虽然可以成功请求，但是 依然会抛出一个警告，被建议去指定证书

超时设置

​ 网页的请求操作是 IO型 的，这意味可能会由于各种原因，例如网络状态不好或服务器出现高延迟等等使得 get 页面时需要等待很长时间，甚至是无法收到响应而报错，对此为了拒绝长时间等待一个响应，设置超时时间 timeout 参数指定一个等待时间值，但在此时间内无法得到结果则主动抛出异常

python

​ timeout 参数的值单位是秒，这个值包含了 连接（connect）和 读取（read）两个阶段的时间总和，因此也可以分别对这两个阶段的超时单独指定通过元组的方式

python

​ 在不设置 timeout 的情况下，默认值是 None，也就是死等，不等到结果不罢休，直到最后报错

python

​ 除了自带的认证方式，还有其他的认证如 OAuth 认证，不过需要安装对应的 oauth 的包

Code

代理设置

​ 代理设置是反爬的一种策略之一，因为通常网站的后台会对网站的流量进行检测，如果一个时间段内频繁的请求页面可能会被检测到，从而网站会要求进行验证码的验证，或是跳回登录页面认证，又或者是将请求的 ip 加到黑名单中禁封一段时间等等，这样导致爬取程序就会停止，为了解决这个反爬策略，通过设置参数 proxies 设置代理 ip
