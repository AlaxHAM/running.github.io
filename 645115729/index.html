数据类型

​ Redis 支持五种数据类型：string（字符串），hash（哈希），list（列表），set（集合）及 zset (sorted set：有序集合)

​ 其中， string 类型是二进制安全的，所以在 redis 中 string 可以存储任何类型，比如序列化对象或是图片二进制字节，且 string 最大能存储 512M

​ redis 中默认所有的命令都是大写，但是小写也没有问题

string

​ 注意，string 类型设置时，普通值是不用自己加双引号的，但是如果值中间有空格，还是需要自己加上双引号

​ 设置指定 key 的值： set key value，设置多个 key-value 可用：mset key value [key value]

Code

​ 获取指定 key 的值： get key，设置多个 key-value 可用：mget key [key]

Code

​ 获取 key 中子字符串（类似切片操作）： getrang key start end

Code

​ 将指定 key 的值修改成 value： getset key value

Code

​ 如果指定的 key 不存在，则会创建这个 key，结果就和使用 set 一样

​ 如果 key 不存在，设置 key： setnx key value，如果 key 存在，则不做 value 更新操作

Code

​ 对 key 的值，递增 +1： incr key，前提是这个 key 的值可以转为数字，比如字符串的 1，如果这个 key 没有设置，则会创建这个 key 并赋值为 1

Code

​ 对 key 的值，递减 -1： decr key

​ 对指定 key 的值，减去给定的量： decrby key decrement

Code

​ 设置一个 key 的失效，超时后会自动删除： setex key seconds value

​ 获取 key 的 长度： strlen key

​ 删除 key： del key

hash

​ 是一个 string 的 filed 字段 和 value 的映射表，其中每一个 hash 都可以存储 2³² - 1 个键值对，大约 40 多亿

​ 在哈希表 key 中设置一对键值对： hset key field value，如果要设置多对键值对，使用 hmset

​ 获取哈希表 key 中的指定字段值： hget key field，获取多个字段使用 hmget

Code

​ 获取哈希表 key 里的所有 value： hvals key，获取所有的 field 和 value 使用 hgetall key，获取所有 filed 使用 hkeys key

Code

​ 判断哈希表 key 的指定 field 是否存在： hexists key field

Code

​ 获取整个哈希表的长度： hlen key

​ 判断 field 是否存在，不存在则设置该字段的 value： hsetnx key field value，同样如果存在，则不会更新

​ 指定 field 的 value 增加指定的数值： hincrby key field increment

​ 删除哈希表 key 中指定 field： hdel key field，可以指定多个字段，这样会一起进行删除，当哈希表中没有数据时，会返回一个为空结果，且这个表会被删除

Code

list

​ 是一个简单的字符串列表，但是允许在列表的两端进行操作，一个列表同样可以 2³² - 1 个元素

​ 在列表 key 左端插入指定元素： lpush key value，放入多个元素，则是依次后面添加，使用空格隔开

Code

​ 从列表 key 左端将元素移除： lpop key

Code

​ 同理，也可以在列表 key 的右端记性元素的添加和移除，分别是 rpush 和 rpop

Code

​ 获取列表 key 中指定范围内的所有元素，即使元素长度不满足范围，也不会出现错误： lrange key start end

Code

​ 通过索引获取列表 key 中的元素： lindex key index

Code

​ 通过索引来设置列表中的元素： lset key index value

​ 对列表进行裁剪，只留下索引在指定范围内的所有元素： ltrim key start stop

​ 从列表 key 的左端移除移除第一个元素，如果没有元素则阻塞住列表，同时等待超时时间，超时就结束，或者在超时时间内列表出现元素，则将该元素从左端移除：blpop key1 [key2] timout

​ 同理，从列表 key 的右端移除元素，也可设置超时： brpop key1 [key2] timout

​ 给已存在的列表末尾添加元素： rpushx key value，如要要在头部则是 lpushx，且如果列表不存在不会进行创建和元素的添加操作

​ 对于 list 类型来说，如果为空，并不会删除，所以如果要删除创建的列表，通过使用 del key

set

​ redis 的 set 是一个 string 类型的无序集合，其实通过 hash 表实现的，即添加，删除，查找都是 O(1)，同样可存储 2³² - 1 个元素

​ 向一个集合添加一个或多个元素： sadd key member1 [member2]

​ 获取一个集合中的所有元素： smembers key

​ 获取一个集合的元素个数： scard key

Code

​ 因为这里是添加了三个元素，所以看不出是无序，可以在增加几个元素，就可以明显看出是无序的

​ 判断一个元素是否在集合中： sismember key member

Code

​ 从集合中随机 移除并返回 一个或多个元素：spop key [count]，如果不给参数 count，则默认从集合中随机移除一个元素并返回，给出 count 就从里面移除 count 个元素。如果要移除所有，count 可以大于 scard 统计的总个数，如果集合为空，则会删除这个集合

Code

​ 从集合中 随机获取 一个或多个元素： srandmember key [count]，和上面的 spop 类似，但是并不会将元素从集合中删除

​ 从集合中 直接移除 一个或多个元素： srem key member1 [member2]

​ 返回两个集合的 交集（即共有部分）： sinter key1 [key2]

​ 返回两个集合的 并集（即两个集合所有元素，重复元素只出现一次）： sunion key1 [key2]

​ 返回两个集合的 差集： sinter key1 [key2] ，返回结果是 key1 - key2，如果是 sinter key2 [key1] 就是 key2 - key1

Code

sorted set

​ 有序集合基本和 set 是一致的，不同的是除了元素是有顺序之外，每一个元素都会 关联一个 double 类型的分数，并通过这个设置的分数来进行从小到大排序

​ 向有序数组中添加元素，注意元素前面需要加上分数： zadd key score1 member1 [score2 member2]，如果元素存在则会更新设置的 score

Code

​ 返回有序数组中指定元素所设置的分数： zscore key member

Code

​ 通过索引(从 0 开始)，返回指定范围内的有序元素： zrange key start stop，如果要获取某一个元素的索引，可以使用 zrank key member，如果需要从大到小排序则是使用 zrevrange key start stop

Code

​ 除了索引获取元素，还可以通过设置的分数，去获取指定范围内的元素： zrangebyscore key min max，同样如果要将结果从大到小排序，使用 zrevrangebyscore key min max

Code

​ 给指定的元素的分数增加指定的值： zincrby key incrme member

Code

​ 从有序数组从移除一个或多个元素： zrem key member [member]

​ 从有序数组中通过指定分数范围移除一个或多个元素： zremrangebyscore key min max
